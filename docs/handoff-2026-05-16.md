# Session Handoff — 2026-05-16

## What Was Done

### TypeScript compile fixes
- Added `thread_title_updated` to `StreamEvent` union in `editor-client/src/contracts/task-contracts.ts`
- Added `updateThreadTitle` to `ControllerUI` interface, `extension.ts` `ui` object, and controller test mock
- Required rebuilding `editor-client` dist before `vscode-extension` typecheck would pass (extension types off compiled output)

### Diff card persistence across reloads
- `run_inline_change` now calls `store.append_message` with a `diff_card` message before broadcasting `diff_ready`
- Before this fix, the diff card disappeared on panel reload since it was only shown via SSE and never written to the thread DB

### Diff card resolved state persistence
- Added `ChatThreadStore.resolve_diff_card(inline_task_id, resolution)` — scans all threads for the message by `task_id`, patches `metadata.resolved`
- Called from the promote and discard API routes after the orchestrator call
- `chat.js` checks `metadata.resolved` when rendering a `diff_card` and shows "✓ Applied" / "✗ Discarded" label instead of buttons

### Logging improvements
- `main.py`: Force-attaches handlers directly on the `agentd` logger (stdout + file) so `--reload` doesn't suppress them (uvicorn owns root logger before `basicConfig` runs)
- Log file: `.agentd/agentd.log` relative to uvicorn CWD — tailable regardless of how the server was started
- Configurable via `CRUCIBLE_LOG_FILE` env var

### Thinking bubble improvements during ToolLoop
- Added `"Making changes…"` thinking broadcast at the start of `run_inline_change` (was stuck on "Drafting change plan…" for the entire ToolLoop duration)
- Controller's SSE handler now handles `tool_call` events → `updateChatThinking("Reading routes.py — <thought>")` 
- Controller handles `patch_applied` → `updateChatThinking("Patch applied…")`
- `tool_call` broadcast payload now includes `args` dict (added in `loop.py`) so filename can be extracted

### Inline diff viewer fix
- "Shadow workspace unavailable" error fixed — the old `openDiffForFile` required `this.session`/`this.latestResult` (task review path)
- New `openInlineDiff(relativePath, shadowPath)` method uses `temp_path` from `diff_entries` metadata directly
- `chat.js` passes `data-shadow-path` on view buttons; `ViewDiffFileHandler` type updated to `(relativePath, shadowPath) => Promise<void>`

### Thinking log persistence
- Every agent turn now accumulates a `thinking_log: list[str]` collecting all `chat_agent_thinking` messages and ToolLoop tool calls
- Stored in message `metadata.thinking_log` for both QA responses and diff cards
- `chat.js` renders a collapsed `<details>Show thinking (N steps)</details>` section under any message that has it
- Persists across reloads — visible in thread history
